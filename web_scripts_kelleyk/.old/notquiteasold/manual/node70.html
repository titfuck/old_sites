Updating this manual

This manual is long and contains lots of information. Probably in no time, parts of this manual will become outdated. Hopefully it has served some use to you. As Netgeek, you should update this manual whenever possible so that future generations will not be lost.

This manual is written with LATEX. It is just a collection of text files, one for each section. Even without familiarity of all the powerof LATEX, you should be able to open up a section file and make changes using emacs, vi, or some other editor that does not add special characters.

LATEXwas chosen because it exists on Athena, is very powerful, and is one of the most widely used typesetting programs on the planet.

This document has a complicated table of contents, section numbering scheme, and figure numbering scheme. Surprisingly, the LATEX document that generates it does not! It is all done automatically, which is one of the reasons why LATEXis a phenominal program.

The source files of this document are located in the tdchouse locker on athena, in the .accounts/NET_GEEK/manual/ directory. The main file of the document is called manual.tex, and a short script called compile in the same directory should be run whenever the manual is updated. Running this program will generate a Postscript file manual.ps, which looks just like the document you are reading now.

SIPB maintains an Inessential Guide to LATEXon their website. It contains more than enough information to get you going with LATEX, which should also be the program of choice for writing any technical papers in your MIT career.
